v1.0 Deployment Guide
Bagian 1
Persiapan di Laptop
Sebelum mulai deploy, pastikan laptop kamu sudah terinstal semua tools yang dibutuhkan.
1
Install Git
  • Download Git dari git-scm.com
  • Klik Next-Next-Next sampai selesai (default sudah benar)
  • Buka CMD atau PowerShell, ketik perintah berikut untuk verifikasi:
git --version

Jika muncul versi seperti git version 2.x.x, berarti Git sudah terinstal dengan benar.

2
Install Node.js
  • Download Node.js dari nodejs.org (pilih versi LTS)
  • Klik Next-Next-Next sampai selesai
  • Buka CMD, ketik perintah berikut untuk verifikasi:
node --version

Pastikan versi yang muncul adalah versi 18 atau lebih baru (contoh: v20.x.x).

3
Install Bun

Bun adalah package manager yang lebih cepat dari npm. Install via CMD:

npm install -g bun

Bun digunakan untuk menginstall dependensi dan menjalankan skrip di project Habit Tracker.

4
Install Vercel CLI

Vercel CLI memungkinkan kamu deploy langsung dari terminal. Install via CMD:

npm install -g vercel

Setelah terinstal, kamu bisa menjalankan perintah vercel dari mana saja di terminal.

Catatan Penting
Setelah semua tools terinstal, restart CMD/PowerShell agar perubahan PATH dikenali oleh sistem.
Bagian 2
Buat Database di Turso
Turso menyediakan database SQLite berbasis cloud secara gratis. Kita akan membuat database dan mengambil kredensial koneksi.
5
Buka Turso dan Login
  • Buka browser, pergi ke: https://turso.tech
  • Klik tombol "Login" atau "Start Free"
  • Login menggunakan akun GitHub: anandategarch
  • Izinkan akses jika diminta oleh Turso
6
Buat Database Baru
  • Di dashboard Turso, klik tombol "Create Database"
  • Isi Database Name dengan: habit-tracker-db
  • Pilih Region yang terdekat: Singapore atau Asia
  • Klik "Create"
  • Tunggu beberapa saat sampai database selesai dibuat (biasanya kurang dari 30 detik)
7
Ambil Koneksi String

Kredensial ini dibutuhkan agar aplikasi bisa terhubung ke database Turso:

  • Di dashboard Turso, klik database habit-tracker-db yang baru dibuat
  • Klik tab "Settings"
  • Cari bagian "Connection string"
  • Copy URL yang dimulai dengan libsql://habit-tracker-db-...
  • Scroll ke bawah, copy juga "Auth Token"
Simpan Kredensial dengan Aman
Simpan kedua nilai tersebut di tempat yang aman (misalnya Notepad). Kita akan memasukkannya ke Vercel pada Langkah 12.
Bagian 3
Buat Repository GitHub
Kode project perlu disimpan di GitHub agar Vercel bisa mengambil dan men-deploy secara otomatis.
8
Buat Repository di GitHub
  • Buka browser ke: https://github.com/new
  • Isi Repository name: habit-tracker
  • Pilih "Public"
  • JANGAN centang "Add a README file"
  • Klik "Create repository"
9
Setup Project di Laptop

Buka CMD atau PowerShell, lalu jalankan perintah berikut satu per satu:

cd Desktop git clone https://github.com/anandategarch/habit-tracker.git cd habit-tracker bun install bun run db:push

Perintah bun install menginstall semua dependensi, dan bun run db:push membuat tabel-tabel di database lokal.

10
Push Kode dari Sandbox

Jika kode sudah ada di GitHub (di-push dari development), langsung lanjut ke Langkah 11.

Jika belum, copy seluruh folder project ke laptop, lalu jalankan:

cd habit-tracker git add -A git commit -m "initial commit" git push -u origin main

Pastikan tidak ada error saat push. Jika diminta login GitHub, ikuti petunjuk di terminal.

Bagian 4
Deploy ke Vercel
Sekarang saatnya men-deploy aplikasi ke Vercel agar bisa diakses dari internet!
11
Deploy via Vercel CLI

Buka CMD di folder project (habit-tracker), lalu jalankan:

vercel

Akan muncul beberapa pertanyaan. Ikuti panduan berikut:

  • Link to existing project? Pilih N (project baru)
  • Project name? Ketik habit-tracker
  • Framework? Akan terdeteksi otomatis: Next.js
  • Pertanyaan selanjutnya: tekan Enter untuk semua default

Ini adalah deploy pertama (preview). Database belum terhubung, jadi halaman mungkin menampilkan error. Itu normal.

12
Set Environment Variables di Vercel

Agar aplikasi bisa terhubung ke database Turso, kita perlu menambahkan environment variables:

Tambahkan DATABASE_URL:

vercel env add DATABASE_URL
  • Pilih: Production, Preview, Development (pilih semua)
  • Paste URL Turso: libsql://habit-tracker-db-xxxxx.turso.io

Tambahkan DATABASE_AUTH_TOKEN:

vercel env add DATABASE_AUTH_TOKEN
  • Pilih: Production, Preview, Development (pilih semua)
  • Paste token dari Turso yang sudah dicopy di Langkah 7
13
Push Database Schema ke Turso

Sekarang kita perlu membuat tabel-tabel database di Turso (bukan di lokal):

  • Pastikan file .env di laptop sudah berisi URL Turso:
DATABASE_URL="libsql://habit-tracker-db-xxxxx.turso.io" DATABASE_AUTH_TOKEN="token-anda-dari-turso"

Lalu jalankan perintah berikut untuk push schema ke Turso:

npx prisma db push

Perintah ini akan membuat semua tabel yang didefinisikan di schema.prisma langsung ke database Turso di cloud.

14
Deploy Ulang ke Production

Sekarang semuanya sudah siap. Jalankan deploy production:

vercel --prod

Tunggu sampai proses selesai (biasanya 1-2 menit). Vercel akan memberikan URL seperti:

https://habit-tracker-xxxx.vercel.app

Buka URL tersebut di browser. Aplikasi Habit Tracker kamu sudah online!

Bagian 5
Selesai! dan Tips Penting
✓
Selamat! Aplikasi Sudah Online
Aplikasi Habit Tracker kamu sekarang online secara permanen! Buka URL dari Vercel di browser mana saja. Data tersimpan di Turso (cloud), aman dan permanen. Bisa diakses dari HP, laptop, atau device lain.
1
Backup Rutin
Setiap minggu, klik "Backup JSON" di halaman Settings untuk menyimpan salinan data kamu secara lokal.
2
Export CSV
Klik "Export Semua CSV" untuk mengunduh semua data dalam format yang bisa dibuka di Excel atau Google Sheets.
3
Update Kode
Setiap ada perubahan kode, jalankan git push lalu vercel --prod untuk memperbarui aplikasi yang live.
4
Free Tier Limits
Vercel gratis: 100GB bandwidth per bulan. Turso gratis: 9GB storage. Cukup untuk penggunaan personal.
Troubleshooting
Masalah yang Sering Terjadi
Error database / koneksi gagal
Cek DATABASE_URL dan DATABASE_AUTH_TOKEN di dashboard Vercel. Pastikan keduanya sudah benar dan cocok dengan yang ada di Turso dashboard.
Schema mismatch / tabel tidak ditemukan
Jalankan npx prisma db push lagi di laptop untuk memperbarui schema database di Turso.
Build failed di Vercel
Jalankan bun run lint di laptop untuk mengecek error. Pastikan semua dependensi terinstall dengan bun install.
